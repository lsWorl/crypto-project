# X25519（椭圆曲线 Diffie-Hellman）模块说明

文件：`src/x25519.c`，头文件：`include/crypto/x25519.h`

概述
- 本模块封装了 X25519 密钥交换及基于共享密钥的会话密钥派生，底层依赖 libsodium 中的 `crypto_scalarmult_curve25519` / `crypto_kx` 等函数。
- X25519（Curve25519 标量乘法）用于 Diffie-Hellman 密钥交换，提供 32 字节公私钥和共享秘密。

主要函数与作用
- `x25519_init()`：初始化 libsodium（调用 `sodium_init()`）。
- `x25519_keypair(public, secret)`：生成密钥对（调用 `crypto_kx_keypair` 或相应函数）。
- `x25519_public_key(public, secret)`：由私钥计算公钥（调用 `crypto_scalarmult_curve25519_base`）。
- `x25519_exchange(shared, my_secret, their_public)`：执行 raw X25519 标量乘法以计算共享秘密（调用 `crypto_scalarmult_curve25519`）。
- `x25519_client_session_keys(rx, tx, client_pub, client_sec, server_pub)` / `x25519_server_session_keys(...)`：在共享秘密基础上用 HKDF-SHA256 派生会话密钥对（rx/tx），并保证客户端/服务端对称关系（info 字符串在客户端/服务端互换）。

设计要点
- 会话密钥派生：
  - IKM = X25519(shared_secret)
  - salt = client_public || server_public（将对方公钥与自身公钥绑定到派生输入，防止重放/跨会话混淆）
  - info 用于区分发送/接收密钥（例如 "x25519-client-tx" / "x25519-client-rx"），确保派生出的 tx 与 rx 互不相同。
- 通过适当交换 info 字符串，保证 server.tx == client.rx、server.rx == client.tx。

安全与实现注意事项
- 由于 libsodium 已对 Curve25519 实现做了完善的输入验证与侧信道防护，直接依赖该库是安全的前提；确保链接并正确初始化 libsodium。
- 共享秘密应在使用后通过 `sodium_memzero` 等安全擦除函数清除。
- X25519 不提供认证；若需防止中间人，应在签名或证书体系下进行密钥认证（例如使用静态签名或双向认证）。

测试
