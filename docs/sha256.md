# SHA-256 模块说明

文件：`src/sha256.c`，头文件：`include/crypto/sha256.h`

概述
- 本模块实现了 SHA-256 摘要算法，遵循 FIPS 180-4 规范。
- 输出为 32 字节（256 位）消息摘要。

主要函数
- `void sha256(const byte *input, size_t input_len, byte *digest)`
  - 输入：任意长度消息及其长度
  - 输出：32 字节摘要写入 `digest`
- `void sha256_print(const byte *digest)`
  - 将 32 字节摘要以小写十六进制打印到 stdout

实现细节与原理
- 常量：初始哈希值 `sha256_initial_hash[8]` 与轮常量 `sha256_round_constants[64]` 直接来自 FIPS 规范。
- 填充（padding）：实现 `sha256_pad`，按 FIPS 要求在消息尾添加 0x80 后续 0 填充，并在末尾追加 64 位的大端消息长度（比特数）。
- 消息分块：按 512 位（64 字节）分块处理；每块进行消息调度生成 64 个 32 位字 `w[0..63]`。前 16 个字通过大端字节组装得来；后 48 个通过 SIG0、SIG1 宏迭代得到（对应规范中的小写 sigma）。
- 压缩函数：实现 `sha256_compress`，初始化 a..h 工作变量，进行 64 轮主循环，使用 EP0/EP1（大写 Sigma）、CH、MAJ 等宏计算中间量，并将结果累加回哈希状态。
- 结束：所有块处理完后，将 8 个 32 位哈希拼成大端字节流写入 `digest`。

性能与复杂度
- 时间复杂度：O(n)（n 为消息字节数），每 64 字节块执行固定 64 轮操作。
- 内存：使用常量轮表与单个 64*4 字节工作数组，内存占用小。

安全与注意事项
- 该实现为教学/自实现版本：虽然遵循规范，但在生产环境中应优先使用经过审计的库（如 OpenSSL、libsodium）。
- 未实现抗时序侧信道保护；对密钥材料的处理处也未特别做常量时间设计。
- 填充、字节序与大端转换处需小心，已有实现采用大端约定与按字节组装，测试向量（见 `test/test_sha256.c`）用于校验。
